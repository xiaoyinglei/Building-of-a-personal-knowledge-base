方案 B+：强可靠性的双速度知识平台

主目标不是“最聪明”，而是“先可靠，再聪明”

顶层原则

可靠性优先级高于自主性。 Agent 不是默认一路放权，它只能在证据足够、边界清晰时扩展检索和推理。图谱与 GraphRAG 是增强层，不是主答案链路的单点依赖。

左侧：主执行链路
文件/网页/笔记接入
解析与标准化
元数据索引 + 全文索引 + 向量索引
检索编排器
回答合成器 + 引用校验
用户答复
右侧：增强能力
模型网关

云模型优先，本地模型兜底

深度研究 Agent

多轮改写查询、扩展证据、对比结论

图谱/GraphRAG

主题网络、实体关系、跨文档桥接

知识沉淀层

摘要、对比页、专题页、引用索引

两条查询路径

Fast Path

快路径

面向简单检索和直接问答。默认 1 次检索编排，1 次重排，1 次回答生成。

目标：快、稳、引用清晰。

Deep Path

深路径

面向跨文档比较、冲突识别、研究型问题。允许多轮检索、证据补充、图谱辅助扩展。

目标：深、全、明确不确定性。

主数据流

从资料到可复用知识

原始文件 → 结构化文档对象 → chunk / entity / summary → 检索与回答 → 半自动沉淀建议 → 专题页/对比页

也就是说，回答不是终点，沉淀才是资产累计的起点。

可靠性护栏

必须有

  • 每个答案都有引用证据
  • 云/本地/联网检索都有策略门控
  • 图谱结果只能增强，不能覆盖原文证据

必须避免

  • 让 Agent 自由发挥到脱离证据
  • 让 GraphRAG 直接主导答案
  • 把本地兜底做成完全不同的一套系统
