查询执行流程与可靠性机制

先判断问题类型，再决定走快路径还是深路径；所有路径都必须回到证据校验

统一入口

用户问题 → 任务分类器 → 快路径 / 深路径 → 回答合成 → 引用校验 → 返回结果

Fast Path
  1. 问题标准化与元数据过滤
  2. 全文 + 向量 + 标签混合召回
  3. rerank 形成证据包
  4. 生成带引用回答
  5. 做 claim-citation 对齐校验

适用：直接问答、单点事实、单文档定位

退出条件：证据稀疏、问题明显跨文档、存在冲突或歧义时，升级到深路径

Deep Path
  1. 问题拆解成 2-5 个子查询
  2. 多轮检索与查询改写
  3. 按文档/观点/时间维度建立证据矩阵
  4. 必要时用图谱做扩展召回
  5. 输出结论 + 分歧 + 不确定性

适用：跨文档比较、观点冲突、研究型问题

约束：必须有轮数上限、token/时间预算、证据阈值

任务分类规则

Route to Fast

快路径信号

问题短、目标明确、限定在单资料或单主题、历史上已有高相似查询。

Route to Deep

深路径信号

出现“比较、冲突、为什么、趋势、多个来源、综合判断”等模式，或首轮证据不足。

Web Search Gate

联网补充条件

本地知识库证据不足，且问题允许公网补充；外部资料必须与内部资料分开标注。

GraphRAG 介入时机

只在第二轮以后增强召回

规则：先做原文混合检索；只有当首轮证据指向若干实体、主题或关系，但覆盖不完整时，才让图谱扩展候选集合。

好处：GraphRAG 帮你“找漏掉的关联”，但不会取代原文证据的主导权。

可靠性护栏

必须执行

  • 回答前做 claim-citation 对齐校验
  • 区分“结论”“证据”“推断”三层输出
  • 证据冲突时显式展示冲突，而不是强行统一
  • 证据不足时允许回答“暂不足以判断”

必须限制

  • 深路径轮数无限增长
  • 让 Agent 自己决定是否引用
  • 把外部网页内容和内部资料混成一个证据池
  • 为了流畅而掩盖不确定性
我建议的默认输出结构

结论：直接回答你的问题

依据：列出关键引用与出处

差异/冲突：如果不同文档说法不同，单独列出来

不确定性：说明缺口在哪里

沉淀建议：是否生成对比页或专题页
